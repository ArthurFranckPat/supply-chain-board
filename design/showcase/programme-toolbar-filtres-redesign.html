Gestion des filtres — Programme toolbar

3 pistes pour désengorger programme-toolbar.tsx : le débordement vient de tout empiler sur une seule rangée nowrap-optionnelle (mode + fenêtre + statut + recherche + portée + refresh + actions). Redimensionne chaque cadre (poignée en bas à droite) pour voir le comportement à l'étroit.

Convenu — architecture sûre

Deux rangées figées : Commande / Filtres

Retour à une séparation en 2 rangées fixes (jamais de wrap possible) : rangée 1 = identité de page (mode, fenêtre, refresh, actions) ; rangée 2 = tout ce qui filtre les données (statut, recherche, portée), fond légèrement teinté --surface-soft pour la distinguer visuellement sans ajouter de bordure lourde.

Coût : +40px de hauteur verticale permanente. Gain : zéro calcul, zéro JS — chaque rangée a moitié moins d'éléments donc ne wrap jamais, même sur écran 13″. C'est l'inverse de la fusion faite en e54b752 ; on la refait mais avec un partitionnement identité/filtre plus net (avant : statut seul était descendu, pas recherche/portée).

Consolidation — réutilise le pattern Actions existant

Un seul déclencheur « Filtres » (statut + recherche + portée)

La rangée reste unique. Statut / recherche / portée sortent de la rangée et vont dans un panneau déroulant — même mécanique <details> que le menu Actions actuel (zéro nouvelle dépendance). Le déclencheur porte des points colorés (statuts actifs) + le nombre de filtres actifs, pour ne pas perdre le signal au clic.

Coût : changer le statut demande 1 clic de plus qu'aujourd'hui (état masqué par défaut). Gain : la rangée perd 3 éléments variables (statut×3 chips + recherche + portée) → largeur quasi constante quel que soit le mode. Cohérent avec l'existant : même geste que « Actions », l'utilisateur n'apprend rien de neuf.

Novateur — collapse mesuré, pas de perte visuelle tant que ça rentre

Priorité adaptative : « +N filtres » seulement si nécessaire

Tout reste visible en ligne, identique à aujourd'hui, tant qu'il y a la place. Un ResizeObserver mesure la rangée ; en dessous d'un seuil, les éléments de plus basse priorité (Suggéré → Portée → Recherche → Statut restants) se replient un par un dans une pilule +N filtres en fin de rangée, qui ouvre un panneau avec les éléments masqués. Priorité fixe : Mode / Fenêtre / Refresh / Actions ne se replient jamais.

Coût : le plus de code (mesure + réordonnancement dynamique), et l'état masqué change avec la largeur (moins prévisible qu'un simple menu fixe). Gain : zéro changement visuel/comportemental sur grand écran (le cas actuel qui marche reste identique) ; dégrade proprement au lieu de wrapper sur petit écran/sidebar ouverte.
