Philosophie : le dot N° porte le statut en pré-attentif (8px, couleur pleine). Le verdict devient du texte teinté — pas de fond, pas de pill. Les filets verticaux créent des « colonnes de registre » qui guident l'œil horizontalement.
+ Très aéré, zéro bruit de fond · + Le dot N° est scannable verticalement (colonne étroite) · − Les filets verticaux ajoutent du bruit structurel · − Le verdict texte sans fond est moins visible que le pill sur 50 rows
Variation B Le signal

Status-first. Les 3 premières colonnes (N° + Commande + Article) ont un fond teinté à 4% de la couleur de statut. Le reste de la row reste blanc. La bordure gauche est conservée. Le verdict pill est conservé mais la zone teintée crée un « bloc de signal » à gauche qui attire l'œil avant même de lire.

Variation C Le compact

Densité maximale. Row height réduite de 30%. Article + désignation sur la même ligne (désignation truncatée). Le verdict devient un dot + texte (pas de pill). La barre de progression Qté est retirée. Objectif : 40% de rows visibles en plus sans scroll.

Recommandation

Baseline + emprunt à B : garder la structure actuelle (bordure gauche + tier 2 + verdict pill) mais ajouter le fond teinté à 3-4% sur les 3 premières colonnes pour les rows en retard/bloqué. C'est le meilleur compromis : le baseline a la bonne hiérarchie, le fond teinté de B ajoute le scan vertical qui manque. Le registre (A) est trop dépouillé pour un outil opérationnel. Le compact (C) est à réserver pour un mode « vue dense » toggleable.
